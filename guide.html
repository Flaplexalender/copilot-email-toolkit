Run the Setup Script

The setup script checks your system and installs what's needed automatically.

  1. Open File Explorer (the yellow folder icon on your taskbar) and navigate to the folder where you saved these files.
  2. Find the file called setup.ps1. Right-click on it. In the menu that appears, Run with PowerShell.
    If Windows blocks the script
    If you see a blue "Windows protected your PC" popup, click More info, then Run anyway.

    If that doesn't work, click the Start menu (Windows icon, bottom-left), type PowerShell, click Windows PowerShell to open it, then type:
    cd "C:\path\to\copilot-email-setup"
powershell -ExecutionPolicy Bypass -File setup.ps1
    Replace C:\path\to\ with the actual folder path.
  3. A blue PowerShell window will appear. The script will check for Python. If Python isn't installed, it will open the download page in your browser.
    Critical: When installing Python
    On the Python installer screen, look at the very bottom. There's a checkbox that says:
    ☐ Add Python to PATH

    Check that box — this is the most important step. Then click Install Now at the top.

    After Python finishes installing, close the PowerShell window and run setup.ps1 again from the beginning.
  4. Next, the script checks for VS Code. If it's missing, it opens the download page. Download and install it with all the default options. Then close PowerShell and re-run setup.ps1.
  5. If both are installed, the script will automatically install extensions and Python packages. You'll see green checkmarks (✓) for each one. Just wait — this takes about 30 seconds.
  6. Then the script will start opening browser tabs for the remaining steps. Follow along! After each step, come back to the PowerShell window and press any key to continue.
  1. Open Finder (the blue smiley face icon in your Dock) and navigate to the folder where you saved these files.
  2. Open Terminal: press ⌘ + Space to open Spotlight, type Terminal, press Enter.
  3. In Terminal, navigate to the setup folder. Type:
    cd ~/Downloads/copilot-email-setup
    (Adjust the path if you saved it somewhere else.)
  4. Make the setup script executable and run it: 
    chmod +x setup.sh
./setup.sh
  5. The script will check for Python 3 and VS Code. If either is missing, it will open the download page for you.
    Python on Mac
    macOS usually comes with Python 3 pre-installed. If the script says it's missing, it will open the Python download page. Download the macOS installer and double-click to install.
  6. Once prerequisites are installed, the script installs extensions and packages automatically, then starts opening browser tabs. Follow along with each step below.
What the script installs for you
VS Code Extensions: GitHub Copilot, GitHub Copilot Chat, Python extension
Python Packages: google-auth, google-auth-oauthlib, google-api-python-client

Create a Google Cloud Project

Don't be intimidated — this is just creating a free container for your email access. No billing required.

  1. The setup script opened console.cloud.google.com/projectcreate. If not, click that link.
  2. If prompted to sign in, use the same Google account as the Gmail inbox you want to manage. Click Next, enter your password, click Next again.
  3. You'll see a "New Project" form. In the Project name field, type Email Helper (or any name you like).
  4. Leave Organization and Location as they are (usually "No organization").
  5. Click the blue Create button at the bottom.
  6. Wait 5–10 seconds. A notification bell (🔔) at the top-right will show "Create Project: Email Helper" when it's done. You may also see a "Select Project" notification — click Select Project if it appears.
Don't worry about billing
Google Cloud may ask about billing for paid services, but the Gmail API is completely free for personal use. You won't be charged. If prompted about billing, you can skip it.

Enable the Gmail API

  1. The setup script opened the Gmail API page. If not, click that link.
  2. Important: Check the top-left of the page — next to "Google Cloud" there's a project dropdown. Make sure it says "Email Helper" (or whatever you named your project). If it shows a different project, click the dropdown and select your project.
  3. You'll see a page titled "Gmail API" with a description. Find the big blue Enable button and click it.
  4. Wait a few seconds. The page will refresh and show "Gmail API" with a green "API enabled" status, plus a dashboard. That's it — you're done with this step!

Set Up the OAuth Consent Screen

This tells Google what your app is called and who can use it (just you).

  1. The setup script opened the consent screen page. If not, click that link.
  2. You'll see "OAuth consent screen" with two options: Internal and External. Select the radio button next to External, then click the blue Create button.
    Why "External"?
    "Internal" is only available for Google Workspace (company/school) accounts. Personal Gmail accounts must use "External". Don't worry — you'll add yourself as a test user so only you can access it.
  3. App information page: Fill in these fields:
    • App name: type Email Helper
    • User support email: click the dropdown and select your Gmail address
    • Scroll all the way down to Developer contact information
    • Email addresses: type your Gmail address
    Leave everything else blank. Click Save and Continue at the bottom.
  4. Scopes page: Don't change anything here. Just click Save and Continue at the bottom.
  5. Test users page: This is important!
    • Click + Add Users
    • In the text box that appears, type your Gmail address (e.g., yourname@gmail.com)
    • Click Add
    • You should see your email listed under "Test users"
    Click Save and Continue.
  6. Summary page: Review the info looks right, then click Back to Dashboard. ✅ Done!

Create OAuth Credentials & Download the Key File

This creates the secret key that lets the script access your email. You'll download a small file.

  1. The setup script opened the Credentials page. If not, click that link.
  2. At the top of the page, find the button that says + CREATE CREDENTIALS and click it. A dropdown menu will appear.
  3. In the dropdown, click OAuth client ID.
  4. You'll see a form. For Application type, click the dropdown and select Desktop app.
  5. For Name, type Email Helper (or leave the default). Click the blue Create button.
  6. A popup window appears titled "OAuth client created". You'll see a Client ID and Client Secret. Don't copy those — instead, look for the button that says DOWNLOAD JSON and click it.
    Critical: Rename and move the file
    1. Find the downloaded file in your Downloads folder. It'll have a long name like client_secret_123456.json
    2. Rename it to exactly: google_credentials.json
    3. Move it into the credentials folder inside this setup kit
    The setup script will have opened the credentials folder for you in File Explorer — just drag the file in. In Finder, navigate to the setup kit folder → credentials → drag the renamed file there.
  7. Click OK to close the popup.
Keep this file private
The google_credentials.json file is like a password. Don't share it, email it, or post it anywhere. It stays on your computer in the credentials folder.

Authenticate Your Gmail Account

This connects the script to your actual Gmail. A browser window will open for you to sign in.

  1. If the setup script is still running, it will do this automatically — just press any key when prompted.
    Otherwise, open PowerShell (Start menu → type "PowerShell" → click it) Terminal (⌘+Space → type "Terminal" → press Enter) , navigate to the folder, and run: 
    python gmail_auth.pypython3 gmail_auth.py
  2. A browser window will open showing the Google sign-in page. Enter your email address and click Next. Enter your password and click Next.
  3. You'll see a warning screen: "Google hasn't verified this app". This is normal and expected.
    Click Continue. If you don't see "Continue", click Advanced at the bottom-left, then click Go to Email Helper (unsafe).
    Why the "unsafe" warning?
    Google shows this because your "Email Helper" project isn't a published app — it's a personal tool that only you can use. Since you created it yourself, it's completely safe to continue.
  4. The next screen asks for permission to access your email. You'll see checkboxes for Gmail access. Click Continue or Allow.
  5. The browser will show "The authentication flow has completed." You can close that tab.
  6. Go back to your terminal/PowerShell window. You should see: 
    ✓ Connected as yourname@gmail.com
✓ Total messages: 12,345
✓ Inbox messages: 234

You're all set! Open VS Code and start chatting with Copilot.
    If you see those green checkmarks, you're authenticated! 🎉

Chat Modes: Ask, Edit, and Agent

Copilot Chat has three modes. The mode selector is at the top of the chat panel — look for a dropdown or icons near where you type.

💬

Ask

Answers questions and explains things. Doesn't run code or make changes. Good for "what does this email mean?" type questions.

✏️

Edit

Makes changes to files you have open. Good for tweaking the gmail_helper.py script if you want to customize it.

🤖

Agent

Can read files, write code, run scripts, and take actions. This is what you want for email management — it can actually execute the Gmail functions.

How to Switch to Agent Mode

  1. Look at the top of the Copilot Chat panel. You'll see a dropdown selector that might say "Ask" or "Chat" or show an icon.
  2. Click it. You'll see the three modes: Ask, Edit, and Agent.
  3. Click Agent. The chat panel will update to show the Agent mode.
When to use each mode
Agent mode — Use this 90% of the time. It can read your emails, run searches, create drafts, archive messages — all the real work.
Ask mode — Use when you just want an explanation (e.g., "What does 'OAuth consent screen' mean?") without running any code.
Edit mode — Use if you want to modify the gmail_helper.py file (e.g., add a new function).

About Autopilot vs. Manual Approval

In Agent mode, Copilot will write code and ask you to approve each action before running it. You'll see a "Run" button or a confirmation prompt.

  • Manual approval (default): You click Run or Continue for each step. Safer — you see what's happening.
  • Auto-approve ("autopilot"): Copilot runs each step without asking. Faster for big cleanup jobs. To enable this, look for a toggle or setting icon at the top of the chat, or just tell Copilot: "go ahead and run everything without asking me."
Recommendation
Start with manual approval so you can see what Copilot is doing. Once you're comfortable and trust the process, switch to auto-approve for big batch operations like "archive all promotional emails from the last year."

Troubleshooting

"Python is not recognized" error (Windows)
Python wasn't added to your system PATH during installation.
  1. Go to python.org/downloads
  2. Download the latest version and run the installer
  3. On the very first screen, check "Add Python to PATH" at the bottom
  4. Click "Install Now"
  5. Close and reopen PowerShell, then try again
"Google hasn't verified this app" scary warning
This is normal and expected. Click Advanced (bottom-left of the warning page), then click Go to Email Helper (unsafe). It's safe because you created this project yourself — Google just warns about all unverified personal projects.
"Access blocked: Email Helper has not completed the Google verification process"
You forgot to add yourself as a test user.
  1. Go to OAuth Consent Screen
  2. Click EDIT APP at the top
  3. Click Save and Continue through until you reach "Test users"
  4. Click + Add Users, type your Gmail address, click Add
  5. Click Save and Continue, then try authenticating again
Copilot doesn't show Claude Opus 4.6 in the model list
  1. Make sure you're subscribed to Copilot Pro — check at github.com/settings/copilot
  2. Make sure you're signed into GitHub in VS Code — look for your avatar icon in the bottom-left corner of VS Code. If it shows a person icon without your photo, click it and sign in.
  3. Try restarting VS Code (close and reopen it)
  4. Make sure the Copilot extensions are installed — click the Extensions icon (four squares) on the left sidebar, search for "GitHub Copilot", and make sure it's installed and enabled.
Copilot says it can't access my email or run scripts
  1. Make sure you're in Agent mode (not Ask or Edit). Check the dropdown at the top of the chat panel.
  2. Make sure gmail_helper.py is open in a VS Code tab (click on it in the left sidebar)
  3. Make sure you opened the correct folder in VS Code (File → Open Folder → select the copilot-email-setup folder)
  4. Try starting your message with: "Read gmail_helper.py in my workspace and use its functions to..."
Token expired / "invalid_grant" error
Your authentication token has expired. Open PowerShellTerminal , navigate to the setup folder, and run: 
python gmail_auth.pypython3 gmail_auth.py
Sign in again and it'll refresh your token. Tokens last about 7 days if unused.
"pip is not recognized" error
Try using python -m pip install instead of pip install. On Mac, use python3 -m pip install. If that doesn't work, Python's Scripts folder isn't in your PATH — reinstall Python with "Add to PATH" checked.
Setup script won't run on Mac — "Permission denied"
Run this in Terminal first: 
chmod +x setup.sh
Then try ./setup.sh again. This gives the script permission to execute.
